Guide

検知チューニングの目安

入力ソース切替

USB / CSI / RTSP はプルダウンで切り替えます。RTSP 選択時だけ URL が必須です。

編集の使い方

スナップショット上で ROI、Perspective、Scale、Line A/B を編集します。監視レイアウトに切り替えると編集パネルを小さくしてライブとログを広く表示できます。

ダウンスケール

処理前に画像を縮小して負荷を下げます。小さくするほど軽くなりますが、細い対象や遠距離の対象は輪郭が消えやすくなります。

フレームスキップ

0 なら毎フレーム処理、1 なら 1 枚おきです。CPU 負荷を下げる代わりに、Tracking の滑らかさや Line Crossing の通過時刻精度は少し落ちます。

最小輪郭 / 最大輪郭

小さすぎるノイズや大きすぎる誤検知を抑える基本パラメータです。デバッグモードでは輪郭サイズも表示されます。

ウォームアップ

起動直後の背景学習に使うフレーム数です。この間は検知結果が安定しにくいので、照明変化が大きい環境では少し長めが安全です。

しきい値

背景差分後の二値化強度です。上げるとノイズは減り、上げすぎると対象も消えます。

背景履歴 / 背景しきい値

背景モデルの追従速度と感度に効きます。環境変化が大きい場合は履歴を短めにします。

Blur / Morphology

Blur は細かいノイズ低減、Morphology は穴埋めや小片除去に有効です。重い場合は両方オフで比較できます。

Open / Dilate

Open は小さなゴミ除去、Dilate は検知領域の膨張です。やりすぎると輪郭が太ります。

追跡距離 / 見失い許容

フレーム間の紐付け条件です。速い被写体では距離を少し広げると安定しやすくなります。

Line Crossing

Line A と Line B の通過時間差で速度を算出します。distance_m には2本間の実距離を入れます。

ppm / px/s

Scale 未設定時は px/s 表示です。実速度にしたい場合は 2点と既知距離を入れて ppm を決めます。
